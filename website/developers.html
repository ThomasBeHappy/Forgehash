Audience

Developers

Port, integrate, and verify bit-exact digests. Prefer Argon2id or platform APIs for real password storage — ForgeHash is experimental.

Start with B3 ($forgeh$v=1$) unless you are intentionally working on the ForgeHash-X sandbox.

Suggested path

  1. 1
    .NET usage

    Install the package, hash / verify / NeedsRehash, Development vs Interactive profiles.

  2. 2
    Language ports

    Rust, Node.js, Python, C++, PHP under langs/ — same vectors, same encoding rules.

  3. 3
    Official vectors

    Freeze digests for B3 v1. Claim compatibility only after all vectors match.

  4. 4
    Implementing

    Checklist, pitfalls, parallel-lane rules, and the implementers/v1/ pack.

Quick commands

# .NET sample
dotnet run --project samples/ForgeHash.Sample -c Release

# CLI hash / verify (B3)
dotnet run --project src/ForgeHash.Cli -c Release -- hash --algo b3 --password-stdin
dotnet run --project src/ForgeHash.Cli -c Release -- verify "$forgeh$v=1$..." --password-stdin

# Language smoke tests
cd langs/python/forgeh && python -m pytest -q
cd langs/nodejs/forgeh && npm test
cargo test --manifest-path langs/rust/forgeh/Cargo.toml --release

Sandbox (optional)

ForgeHash-X is a separate experimental track. Digests and encoding are not interchangeable with B3.
